## TokenViewer v0.2.7

This release fixes a Codex usage-tracking gap for sandboxed dev environments, improves Kiro model version detection, and adds skill sync filtering.

### 🐛 Fixes
- Fixed Codex usage tracking to also read sessions from `$CODEX_HOME` when a launcher redirects it to a sandboxed runtime home (e.g. Orca), so newer Codex activity is no longer missed. Behavior is unchanged for the default setup where `$CODEX_HOME` isn't set.
- Fixed Kiro model version normalization to correctly detect and label more Claude/GPT/Gemini model variants instead of falling back to the raw internal model ID.

### ✨ Improvements
- Added filtering options to the Skills Git sync sheet.

### 📦 Installation
**Recommended:** Download `TokenViewer.dmg`
- Drag `TokenViewer.app` to `/Applications`
- First launch: System Settings -> Privacy & Security -> Open Anyway
- Requires macOS 14 (Sonoma) or later

**Alternative:** Download `TokenViewer-Installer.pkg`
- Double-click to install -> app is placed in `/Applications`
- The installer updates the existing app in place and opens it after installation
- Requires macOS 14 (Sonoma) or later
