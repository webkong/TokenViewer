## TokenViewer v0.1.8

This release fixes critical token counting bugs in the Claude, Codex, and CodeBuddy parsers, and adds CodeBuddy to the Limits page.

### ✨ Features
- CodeBuddy now appears in the Limits page with live quota display (shares the same API backend as WorkBuddy).
- The app automatically rebuilds parsed data on version upgrade, ensuring parser fixes take effect without manual intervention.
- Models view now aggregates token usage by model name regardless of source, eliminating duplicate entries.

### 🐛 Fixes
- Claude Code parser now deduplicates by message.id, fixing 2–3× token overcounting when the same response produces multiple usage lines.
- New session files are now correctly discovered even when the parent directory mtime hasn't changed (subdirectory-aware glob cache).
- Codex cumulative delta calculation now normalizes input tokens after diffing, preventing phantom resets when cache hit rate fluctuates.
- CodeBuddy parser no longer filters out function_call entries, recovering ~85% of previously missing token data.
- CodeBuddy reasoning_tokens now read from completion_tokens_details instead of prompt_tokens_details.
- Claude parser skips lines without a valid timestamp instead of mis-attributing them to the current time bucket.

### 📦 Installation
**Recommended:** Download `TokenViewer.dmg`
- Drag `TokenViewer.app` to `/Applications`
- First launch: System Settings -> Privacy & Security -> Open Anyway
- Requires macOS 14 (Sonoma) or later

**Alternative:** Download `TokenViewer-Installer.pkg`
- Double-click to install -> app is placed in `/Applications`
- The installer updates the existing app in place and opens it after installation
- Requires macOS 14 (Sonoma) or later
