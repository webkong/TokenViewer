## TokenViewer v0.1.9

This release improves data accuracy and consistency, fixes Kiro usage attribution, and hardens incremental sync.

### 🐛 Fixes
- Kiro IDE token data is no longer double-counted or dumped onto a single day: when the timestamped `devdata.sqlite` exists it is authoritative, and the timestamp-less `tokens_generated.jsonl` is treated as a fallback only.
- Daily/hourly/heatmap aggregation already buckets by your local calendar day; the Models view aggregates by model name across sources.
- Claude conversation counts now reflect real user prompts (main sessions, text turns) instead of every assistant message, fixing ~7× inflated conversation numbers.
- Switching the time range quickly (week ↔ month) no longer lets a superseded query overwrite the current range's data.

### ✨ Improvements
- Kiro CLI history (`data.sqlite3`) now syncs incrementally via an `updated_at` + per-conversation timestamp watermark — faster, compaction-safe, and no longer relies on an unbounded dedup set.
- Incremental file cursors are invalidated when a log file's inode changes (truncation / recreation / rotation), preventing missed reads.
- Unknown models are now logged once instead of silently costing $0, making pricing gaps visible.

### 📦 Installation
**Recommended:** Download `TokenViewer.dmg`
- Drag `TokenViewer.app` to `/Applications`
- First launch: System Settings -> Privacy & Security -> Open Anyway
- Requires macOS 14 (Sonoma) or later

**Alternative:** Download `TokenViewer-Installer.pkg`
- Double-click to install -> app is placed in `/Applications`
- The installer updates the existing app in place and opens it after installation
- Requires macOS 14 (Sonoma) or later
