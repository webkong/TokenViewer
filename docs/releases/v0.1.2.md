## TokenViewer v0.1.2

This release focuses on macOS polish: update flow improvements, safer data recovery, and smoother day-to-day sync behavior.

### ✨ Features
- Auto-sync now runs when opening the menu-bar popover or switching to the dashboard, keeping the UI up to date without a manual refresh.
- About now automatically downloads new macOS releases after an update is detected, while keeping the install confirmation step.
- Data management now includes a rebuild action that clears processed data and sync cursors, then replays raw sources from scratch.

### 🐛 Fixes
- Codex usage entries now preserve the correct model name when the source only exposes `model_provider`, avoiding `unknown` model labels in summaries.
- Update checks and About/Settings copy were aligned with the in-app localization system, removing hardcoded English strings from the update flow.
- The Settings page now keeps release-related information in About only, and the destructive reset-all-data path has been removed.
- macOS launch behavior now opens the main window during development runs, which makes `bash run.sh` behave like an actual app launch.

### 📦 Installation
**Recommended:** Download `TokenViewer-0.1.2-Installer.pkg`
- Double-click to install → app is placed in `/Applications`
- Quarantine attribute is removed automatically — no Gatekeeper warning
- Requires macOS 14 (Sonoma) or later

**Alternative:** Download `TokenViewer.dmg`
- Drag `TokenViewer.app` to `/Applications`
- First launch: System Settings → Privacy & Security → Open Anyway
