## TokenViewer v0.2.10

This release adds support for Kiro CLI's new session format and makes Usage and Skills defaults smarter and safer.

### ✨ New

- Added support for Kiro CLI's new per-workspace session store (`sess_*` directories), fixing usage tracking for recent Kiro CLI sessions that were previously not counted at all.
- Added Skills onboarding: a first-run guide explaining where the shared Skills library lives and how per-agent symlinks control visibility, reopenable anytime from the Skills header.

### 🐛 Fixes

- Fixed the shared Skills library defaulting to `~/.agents/skills`, a path some agents scan on their own — a Skill organized there could become visible to an agent without an explicit symlink. The default is now `~/.tokenviewer/skills`; existing custom paths are unaffected.
- Fixed the software update check silently reporting "Up to date" on network or rate-limit errors instead of surfacing the failure.
- Fixed the automatic update check triggering a Keychain authorization prompt on every app launch; it's now only used for manual "Check Now" checks.

### 🎨 Improvements

- The Usage dashboard now opens on Today when there's already data for today, or Yesterday otherwise, instead of always defaulting to the last 7 days.

### 📦 Installation
**Recommended:** Download `TokenViewer.dmg`
- Drag `TokenViewer.app` to `/Applications`
- First launch: System Settings -> Privacy & Security -> Open Anyway
- Requires macOS 14 (Sonoma) or later

**Alternative:** Download `TokenViewer-Installer.pkg`
- Double-click to install -> app is placed in `/Applications`
- The installer updates the existing app in place and opens it after installation
- Requires macOS 14 (Sonoma) or later
