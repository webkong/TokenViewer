## TokenViewer v0.2.0

This release brings SkillSync into TokenViewer as a native Skills manager, improves Limits navigation and Codex limit parsing, and makes the macOS app available from the Dock.

### ✨ New
- Added a native Skills page for scanning, filtering, organizing, restoring, deleting, and syncing agent skills.
- Added Git sync authentication for skills, including repository, token, username, and email settings.
- Added agent-aware skill ownership so skills linked into multiple agent directories are shown across those agents.
- Added installed-agent detection and shared provider metadata from a single core registry.
- Added a Limits shortcut from the menu bar and moved the Limits tab next to Usage.
- Added Dock icon support; clicking the Dock icon reopens the main panel.

### 🐛 Fixes
- Fixed skills scanning for selected agents, including symlinked skills and per-agent skill directories.
- Fixed duplicate and misleading skill sync toast messages.
- Fixed Skills and Settings inputs keeping focus after clicking outside.
- Fixed global skills path display so `~` is preserved instead of expanding into the full home path.
- Fixed Codex limits reset time parsing for session and weekly windows.
- Fixed Codex subscription expiry display when token claims contain a stale billing period.

### ✨ Improvements
- Refined the Skills page layout with wrapped agent filters, a clearer list header, compact action buttons, and agent badges that stay visible.
- Improved global toast behavior so messages appear once, animate cleanly, and stay above sheets without duplicate overlays.
- Sorted agents with installed providers first across Skills and menu bar settings.
- Updated provider icons and display metadata through the unified provider registry.

### 📦 Installation
**Recommended:** Download `TokenViewer.dmg`
- Drag `TokenViewer.app` to `/Applications`
- First launch: System Settings -> Privacy & Security -> Open Anyway
- Requires macOS 14 (Sonoma) or later

**Alternative:** Download `TokenViewer-Installer.pkg`
- Double-click to install -> app is placed in `/Applications`
- The installer updates the existing app in place and opens it after installation
- Requires macOS 14 (Sonoma) or later
