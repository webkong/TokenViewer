## TokenViewer v0.1.1

Improves Kiro usage tracking and fixes day-dimension aggregation to respect your local timezone.

### 🐛 Fixes
- **Local-time day/hour aggregation** — the heatmap, daily trend, and hourly chart now bucket usage by your local calendar day instead of UTC. Previously, usage between local midnight and the UTC offset (e.g. 00:00–08:00 in UTC+8) was attributed to the previous day.
- **Single Kiro provider** — Kiro IDE and Kiro CLI are now merged under one "Kiro" entry instead of showing a separate `kiro-ide`.

### ✨ Improvements
- **Kiro CLI historical data** — reads `~/Library/Application Support/kiro-cli/data.sqlite3` (`conversations_v2`), so past CLI conversations are tracked even on devices where the older IDE data source stopped writing.
- **Accurate Kiro model breakdown** — real model versions (`claude-sonnet-4.6`, `claude-opus-4.8`, …) are shown per usage; `kiro-agent` now only serves as the fallback bucket for usage that can't be attributed to a specific model.

### 📦 Installation
**Recommended:** Download `TokenViewer-0.1.1-Installer.pkg`
- Double-click to install → app is placed in `/Applications`
- Quarantine attribute is removed automatically — no Gatekeeper warning
- Requires macOS 14 (Sonoma) or later

**Alternative:** Download `TokenViewer.dmg`
- Drag `TokenViewer.app` to `/Applications`
- First launch: System Settings → Privacy & Security → Open Anyway
