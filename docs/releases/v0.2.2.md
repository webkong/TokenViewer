## TokenViewer v0.2.2

This release fixes the Limits dashboard page incorrectly following the menu-bar visibility setting, makes the Skills organize feature work correctly for any agent with a nested skills layout, refines the menu-bar limits card defaults, and adds several missing model brand icons.

### 🐛 Fixes
- Fixed the Limits dashboard page hiding agents that were only hidden from the menu-bar popover — the dashboard now always shows every agent that supports quota tracking.
- Fixed Skills "organize" failing for agents with a nested skills directory layout (e.g. Codex's `.system/<skill>`) when triggered through the bulk-organize path; the fix is now generalized so it works for any agent, not just Codex.
- Fixed the LongCat model showing a generic fallback icon instead of its brand logo.

### ✨ Improvements
- Menu-bar limits cards now default to Claude Code, Codex, Gemini, and Kiro plus any other limits-capable agent detected as installed on your machine, instead of showing all 15 supported agents by default.
- Added brand icons for LongCat, StepFun, Wenxin (Baidu), Hunyuan (Tencent), Llama (Meta), Mistral, Cohere, Yi (01.AI), and Perplexity models across the usage breakdown views.

### 📦 Installation
**Recommended:** Download `TokenViewer.dmg`
- Drag `TokenViewer.app` to `/Applications`
- First launch: System Settings -> Privacy & Security -> Open Anyway
- Requires macOS 14 (Sonoma) or later

**Alternative:** Download `TokenViewer-Installer.pkg`
- Double-click to install -> app is placed in `/Applications`
- The installer updates the existing app in place and opens it after installation
- Requires macOS 14 (Sonoma) or later
