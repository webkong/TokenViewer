## TokenViewer v0.2.3

This release improves the Skills manager, fixes several skill organization edge cases, and makes the main window open reliably when returning to TokenViewer from the Dock or app switcher.

### Fixes
- Fixed global skills being incorrectly narrowed to a single agent when they were re-discovered through agent symlinks.
- Fixed Codex system skills so built-in skills are detected by their agent source instead of by whether they have a manifest file.
- Fixed the built-in skills filter and restored built-in badges in the Skills manager.
- Fixed TokenViewer not reopening its main window when switching back to the app from the Dock or Cmd-Tab.

### Improvements
- Added source-aware skill compatibility so agent-local skills default to the agent where they were found while global skills remain universal.
- Added a skill preview sheet for reading a skill's `skill.md` directly from the Skills page.
- Added clearer agent chips and provider colors in the Skills manager.
- Added a confirmation prompt before organizing built-in skills into the global library.
- Added Dock icon and menu-bar icon visibility controls in Settings.

### Installation
**Recommended:** Download `TokenViewer.dmg`
- Drag `TokenViewer.app` to `/Applications`
- First launch: System Settings -> Privacy & Security -> Open Anyway
- Requires macOS 14 (Sonoma) or later

**Alternative:** Download `TokenViewer-Installer.pkg`
- Double-click to install -> app is placed in `/Applications`
- The installer updates the existing app in place and opens it after installation
- Requires macOS 14 (Sonoma) or later
