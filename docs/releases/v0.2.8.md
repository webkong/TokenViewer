## TokenViewer v0.2.8

This release fixes local-day usage reporting and makes Codex tracking reliable when Codex runs inside Orca.

### 🐛 Fixes
- Fixed daily usage, hourly trends, active-day counts, and the activity heatmap to consistently use the Mac's local calendar day instead of UTC boundaries.
- Fixed Codex usage tracking for Orca sessions even when TokenViewer is launched without Orca's `$CODEX_HOME` environment variable.
- Prevented standard Codex and Orca-mirrored rollout files from being counted twice.

### ✨ Improvements
- Unified usage date ranges and provider timestamp parsing for more consistent time handling across the app.
- Standardized the Today, 7-day, and 30-day ranges as rolling local-calendar periods.

### 📦 Installation
**Recommended:** Download `TokenViewer.dmg`
- Drag `TokenViewer.app` to `/Applications`
- First launch: System Settings -> Privacy & Security -> Open Anyway
- Requires macOS 14 (Sonoma) or later

**Alternative:** Download `TokenViewer-Installer.pkg`
- Double-click to install -> app is placed in `/Applications`
- The installer updates the existing app in place and opens it after installation
- Requires macOS 14 (Sonoma) or later
